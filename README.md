This repository is the companion code for the peer-reviewed article
*"Forecasting Bitcoin Prices and Analyzing Geopolitical Risks:
A Residual Analysis Approach using LSTM"* (Mendes, Bezerra & Rocha,
2026).

It contains everything needed to reproduce the paper end-to-end:
data, notebooks, a modular Python package, unit tests, continuous
integration and trained model weights.

---

## Scientific question

> Can unusually large prediction errors of a univariate LSTM forecast
> of daily Bitcoin closing prices serve as empirical markers of
> geopolitically significant events?

The pipeline is organised in **two phases**:

1. **Quantitative phase.** Fit an LSTM on 2012-2014 BTC/USD Bitstamp
   closing prices, run out-of-sample inference over all 2015 trading
   days and flag every day whose residual falls outside a ±2σ band.
2. **Qualitative + causal-inference phase.** Each flagged day is
   expanded into an asymmetric ``[-3, +7]``-day investigation window,
   merged with a curated qualitative event database
   (``data/external/qualitative_analysis.csv``), and used to compute
   the **Explaining Ratio (ER)** — the share of LSTM outliers that
   admit a contemporaneous geopolitical or institutional explanation.

See [`docs/METHODOLOGY.md`](docs/METHODOLOGY.md) and the technical
reports under [`reports/`](reports/) for the full methodology.

## Installation

The project is packaged as `geo-btc` (version `1.0.0`).

### pip

```bash
git clone https://github.com/Erick-wilding675/Geo-BTC.git
cd Geo-BTC
git lfs install && git lfs pull

python -m venv .venv
source .venv/bin/activate     # Windows: .venv\Scripts\activate
make install                  # pip install -e ".[dev,notebook]"
```

### conda

```bash
git clone https://github.com/Erick-wilding675/Geo-BTC.git
cd Geo-BTC
git lfs pull

make env
conda activate geo-btc
pip install -e .
```

See [`docs/REPRODUCIBILITY.md`](docs/REPRODUCIBILITY.md) for OS-level
notes and deterministic-GPU caveats.

## Reproducing the paper

Run the entire pipeline end-to-end:

```bash
make pipeline
```

…or invoke individual stages:

```bash
# ── Phase 1 — quantitative ─────────────────────────────────────
make preprocess        # scripts/run_preprocessing.py
make train-baseline    # §3  baseline LSTM
make train-tuned       # §4  stacked LSTM
make grid-search       # §5  hyper-parameter search
make train-final       # §6  final model on merged 2012-2014
make residuals         # §6-7 2015 inference + outlier detection

# ── Phase 2 — causal inference ─────────────────────────────────
make merge             # scripts/run_inference_merge.py
make er                # scripts/run_explaining_ratio.py
make plot              # scripts/run_inference_plot.py
# or all three at once:
make phase2
```

Every script accepts `--config` to override the default
[`configs/config.yaml`](configs/config.yaml):

```bash
python scripts/run_residual_analysis.py --config configs/config.yaml
python scripts/run_explaining_ratio.py --threshold 20
```

## Running the tests

```bash
make test     # pytest + coverage   (currently 31 passing tests)
make lint     # ruff check
make format   # ruff format + --fix
```

The `tests/` suite is self-contained — it builds synthetic OHLCV
frames and synthetic event windows on the fly and does **not** depend
on the Bitstamp CSV.
